Challenge 17: The Solomonic pizza cut


Daniel's favourite food is pizza, which he likes to share with his girlfriend, Sara. As she also loves pizza, it is of utmost importance, in order to avoid unnecessary quarrels, to split the pizza into two equally-sized slices. Of course, the pizza contains ingredients such as pepperoni, sausage, pineapple, etc. These ingredients are delicate and cannot be cut*. As you can imagine, both Sara and Daniel must eat the same number of ingredients of each type.

The pizza shop where they order from is not a typical one. They like to decorate their fabulous disks by using shaped ingredients. This spring, it is time for regular polygons!

As you can easily observe, depending on the position, size and shape of the ingredients, it is not always possible to make a straight-line cut into the pizza in such a way that the two slices have the same area and the same number of ingredients of each type. Given a pizza and its ingredients, is there a straight-line cut that makes both Daniel and Sara happy?


*An ingredient is cut if it is split in two slices with positive area.


Input

The first line of the input gives the number of test cases, C, and C test cases follow (1 ≤ C ≤ 50). Each test case has the following format. Cx Cy R
N
IDi Ni,e Ni
Cij,x Cij,y Vij,x Vij,y

Where,
(Cx, Cy) is the center of the pizza and R its radius.
N is a non-negative integer, 0 ≤ N ≤ 10, that tells how many different types of ingredients the pizza contains.
IDi is a string which identifies the i-ingredient.
Ni,e is an integer, 3 ≤ N ≤ 7, that indicates how many edges the ingredients of type IDi have.
Ni is an integer, 1 ≤ Ni ≤ 10, that indicates how many ingredients of type IDi the pizza has.
(Cij,x Cij,y) is the center of the ingredient ij, 1 ≤ j ≤ Ni
(Vij,x Vij,y) is a vertex of the ingredient ij

Output

For each test case, output one line containing "Case #x: M", where x is the case number (starting from 1) and M is the string "TRUE" or "FALSE" depending on whether the pizza can be cut or not.

Sample input

2
0.0 0.0 50.0
1
cheese 4 2
19.298 5.1234 12.2341 7.33
5.345 -9.876 7.083 -2.214
-9.4846 -0.2218 46
2
cheese 4 2
-20.9207 0.1189 -20.9207 -6.0353
-6.4074 7.3172 -6.4074 1.1629
mushroom 4 2
-17.3105 9.3976 -13.234 8.072
10.8611 -1.0212 -8.1733 -4.3604

Sample output

Case #1: TRUE
Case #2: FALSE

Submit & test your code

To test and submit code we provide a set of tools to help you. Download contest tools if you don't have done that already. Then with the challenge token you will be able to test and submit your solution to this challenge.

Challenge token: CHALLENGE_17

To test your program

./test_challenge CHALLENGE_17 path/program

A nice output will tell you if your program got the right solution or not. You can try as many times as you need.

To submit your program to the challenge

./submit_challenge CHALLENGE_17 path/source_pkg.tgz path/program

Note that you first need to solve the test phase before submiting the code. On the submit phase we could give your program harder questions, so try to make your program failsafe.

Remind you can only submit once!. Once your solution is submitted you won't be able to amend it to fix issues or make it faster, so please be sure your program is finished before submitting it.
